      Step 2

      Cloudflare account and Worker URL

      Do this before creating the TickTick developer app. TickTick needs your production callback URL, which comes from the Worker hostname you choose here.

    • Log in to Cloudflare 
      npx wrangler login
    • Confirm or create the D1 database

      wrangler.jsonc already lists a D1 binding named ticktick_sync, with a placeholder database_id. Create a database on your Cloudflare account, then replace the placeholder with the UUID that wrangler d1 create prints — remote migrate/deploy will fail until that ID is yours. 

      npx wrangler d1 create ticktick_sync

      Copy the UUID from the command output into wrangler.jsonc → d1_databases[0].database_id. Local npm run db:migrate:local / npm run dev can work without a real remote ID, but production deploy cannot.

    • Decide your production Worker hostname

      After your first deploy (or from the Cloudflare dashboard), you will have a hostname like ticktick-sync.<account>.workers.dev or a custom domain. Enter it below so later steps can show the exact TickTick callback URL.

      Host only — no https:// and no path.

      • You can finish deploy later

      Knowing (or choosing) the hostname is enough for TickTick registration. Full production deploy happens in Step 8 after secrets and local OAuth.

        Support
        
          
        Common failures
        
          
        
            
              
        
                Symptom
                Likely cause
              
        
                    
            
              
        
                TickTick OAuth fails with a redirect mismatch
                The callback URI is not registered exactly, including http vs https.
              
        
              
        
                /debug/projects returns 401
                The DEBUG_TOKEN Bearer header is missing or wrong.
              
        
              
        
                GitHub webhook returns 401
                GITHUB_WEBHOOK_SECRET does not match the value configured in GitHub.
              
        
              
        
                Azure DevOps webhook returns 401
                The Service Hook basic auth username/password does not match your Worker secrets.
              
        
              
        
                Azure DevOps backfill says org/project must be configured
                You still have placeholder YOUR_ORG / YOUR_PROJECT values in wrangler.jsonc.
              
        
              
        
                Tasks go to the wrong TickTick list
                The source-specific list IDs are wrong, or you intentionally enabled shared-list mode but forgot.
              
        
              
        
                No tasks appear in production after local testing worked
                Production TickTick OAuth has not been completed yet. Local and remote D1 databases are separate.
              
        
              
        
                D1 errors mention missing tables
                Run npm run db:migrate:local or npm run db:migrate:remote.
